# Codex Skills 与 Plugins：安装、调用、验证和卸载

本文面向第一次接触 Codex 扩展能力的职场用户。

## 1. 先理解两个概念

### Skill

Skill 是可复用的工作流说明。一个标准 Skill 是一个目录，至少包含 `SKILL.md`；该文件需要写明 `name` 和 `description`，还可以附带：

- `scripts/`：辅助脚本
- `references/`：参考资料
- `assets/`：模板和素材
- `agents/openai.yaml`：界面信息、调用策略和工具依赖

Codex 可以被明确要求使用某个 Skill，也可以根据 `description` 自动匹配。

### Plugin

Plugin 是分发载体，可以同时包含多个 Skills、连接器、MCP 服务配置、浏览器能力、Hooks 或定时任务模板。需要连接 Gmail、Google Drive、Slack 等外部服务时，通常由 Plugin 提供连接与授权能力。

## 2. 如何判断下载内容能不能直接安装

打开下载目录，检查是否满足：

```text
skill-name/
└─ SKILL.md
```

并确认 `SKILL.md` 开头至少类似：

```yaml
---
name: skill-name
description: 说明它在什么任务下触发，以及不适用的边界。
---
```

只有普通 `.md` 文档、提示词或岗位说明时，它可以作为参考模板使用，但不等于可直接安装的 Codex Skill。

## 3. 仓库级安装

适合只在当前项目中使用。把完整 Skill 文件夹复制到项目的：

```text
项目根目录/.agents/skills/skill-name/
```

示例：

```text
my-office-project/
├─ .agents/
│  └─ skills/
│     └─ review-document-delivery/
│        └─ SKILL.md
└─ 工作文件/
```

Codex 会从当前工作目录向上扫描到仓库根目录中的 `.agents/skills`。

## 4. 用户级安装

适合在多个项目中共用。把完整 Skill 文件夹复制到：

```text
$HOME/.agents/skills/skill-name/
```

Windows 通常对应当前用户主目录下的：

```text
C:\Users\你的用户名\.agents\skills\skill-name\
```

安装后 Codex 通常会自动识别；若没有出现，重启 Codex。

## 5. 使用内置安装器

安装官方精选 Skill 或从其他仓库下载 Skill 时，可直接在 Codex 中调用：

```text
$skill-installer
```

例如：

```text
$skill-installer linear
```

也可以明确告诉安装器要从哪个 GitHub 仓库、哪个目录下载。安装第三方 Skill 前仍需检查许可证和脚本。

## 6. 安装 Plugin

在 ChatGPT 桌面端中：

1. 进入 Codex，或在 ChatGPT 中切换到 Work 模式。
2. 打开 Plugins。
3. 浏览并选择需要的 Plugin。
4. 阅读它包含的 Skills、连接器、Hooks 和权限。
5. 安装；如需外部服务，再按提示连接账号。
6. 新建一个任务验证能力是否可用。

在 Codex CLI 中可输入：

```text
/plugins
```

安装后新开一个会话，再使用它提供的 Skills 或工具。

## 7. 如何调用 Skill

### 明确调用

在提示词中直接点名：

```text
请使用 $skill-name 处理这份文件，并在交付前完成验证。
```

CLI 或 IDE 中也可使用 `/skills`，或输入 `$` 后选择 Skill。

### 自动调用

当任务与 `SKILL.md` 中的 `description` 高度匹配时，Codex 可以自动选择。描述写得越清楚，误触发越少。

## 8. 安装后怎么验收

不要只看“安装成功”，至少做一次最小测试：

1. 新建不含敏感数据的测试文件。
2. 明确点名 Skill 执行一个小任务。
3. 观察 Codex 是否读取了正确的 `SKILL.md`。
4. 检查输出路径、格式、公式、版式或文档内容。
5. 确认原文件没有被意外覆盖。
6. 如有脚本，检查它是否访问网络、读取密钥或删除文件。

## 9. 暂停或卸载

### 暂停某个 Skill

可在 `~/.codex/config.toml` 中加入：

```toml
[[skills.config]]
path = "/path/to/skill/SKILL.md"
enabled = false
```

修改后重启 Codex。

### 卸载

删除你自己安装的 Skill 文件夹即可。删除前确认路径准确，不要误删 Codex 内置目录或其他项目文件。

Plugin 请从桌面端或 CLI 的 Plugins 管理界面卸载；外部服务授权还应在对应服务中单独检查和撤销。

## 10. 本仓库第三方内容的正确用法

`third-party-mit/awesome-niuma-skills/content/` 当前主要是“岗位工作模板库”：

- 普通 Markdown：作为提示词、检查清单或工作流参考。
- 含 `SKILL.md` 的目录：仍应先审查脚本和适用范围，再安装测试。
- ZIP：本次公开快照已排除，避免重复内容和未核验的嵌套项目。

后续经过重构、测试并符合标准的 Skill 会进入独立目录，并提供版本化下载链接；不会直接覆盖原始第三方快照。
